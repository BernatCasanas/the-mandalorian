All about Diamond Engine

Diamong Engine is a lightweight 3D (and future 2D) game development engine. This engine was developed as part of a college assigment. The engine supports almost every editor feature that you could expect from a game development engine, but the most relevant feature is the added support to interpret and work with C# code on a component based GameObject system.

The C# implementation was mainly focused on making it as fast an reliable as possible. This implementation uses the mono runtime working with JIT to execute C# code at runtime. Whit this implementation and the app-domain system, it is also possible to recompile code at runtime to make the iteration time as fast as possible.

Apart from the performance and the reliable dynamic memory managment, the engine is also focused on being visualy atractive to the users. Also new features are being added to make the engine able to generate bigger and better games.

Core Sub-Systems

  • Optimized 3D rendering

    OpenGL rendering was optimized avoiding deprecated methods and systems, and its performance was also improved by rendering as much as possible with retained mode.

  • Camera Culling

    Scene rendering time was reduced with camera culling, camera frustrum and AABB boxes to discard all the meshes outside the camera view. A future improvement will be made by adding space partitioning (a field I made a small study of back in 2018 that can be readed here).

  • Editor Docking

  • Script editor

  • GameObject & Component System

  • Multi-viewport Rendering

  • Memory managment

  • Time Managment

    The time managment system makes it possible to start/stop/pause/step a simulation that will also run other systems (like shaders, particles...). The content played during this simulation will be the exact content played/displayed when the game is compiled and executed as a standalone application.

  • Resource Manager

    Resource managment systems make sure that we don't use more memory than we really need by only loading resources once to memory, and then the system will assign references (pointers) to those loaded resources. This system reduces drasticly the memory usage of the application and the final game.

  • Style Customization

  • Screenshoot System

    A small screenshoot system was added to the editor that allows the user to capture simulation moments or editor frames as .PNG files.

C# Sub-System

As mentioned above, Diamond Engine has a C#/Mono implementation, making it perfect for faster code iterations.

This implementation makes heavy use of extern methods to send and recieve data from C++ and C#, which are connected.

One of the biggest performance optimizations was achieved by sending C++ pointers to the C# objects as UINT64 values, and then, when we need to retrieve some data from C++ to use on C#, we just have to send and cast that pointer UINT64 to C++ and then send the data that we need to C#.

As trivial as this may sound, this small detail reduced considerably the C++/C# communication time, which also made Update times a lot faster.
